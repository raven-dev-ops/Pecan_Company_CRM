# Discovery Spec: Receipt Content and Numbering

Issue link: #6

## Required receipt fields
- Business name, address, phone
- Receipt number
- Date/time (local timezone)
- Line items (name, qty/weight, unit price, line subtotal)
- Subtotal
- Discount (if applied)
- Tax (if enabled)
- Total
- Payment method
- Optional customer summary (name/phone)

## Receipt numbering decision
- Format: six-digit zero-padded numeric string (`000001`, `000002`, ...)
- Prefix: not required for MVP.
- Number generated by database-safe sequence strategy.

## Local storage decision
- Default folder: `%ProgramData%\PecanCRM\receipts\`
- Suggested hierarchy: `%ProgramData%\PecanCRM\receipts\YYYY\MM\`
- Filename: `receipt_<receipt_number>.pdf`

## Regeneration behavior
- Receipt can be regenerated from persisted sale + sale_items data.
- Reprint is allowed even if original PDF file is missing.
